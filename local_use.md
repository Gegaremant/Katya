# Тестирование локальных моделей (Ollama) через SSH-туннель

Этот файл содержит конкретные инструкции по поднятию туннеля к вашему серверу `srv-llm` для использования Ollama через приложение Katya.

## Ваши настройки туннеля (из watchdog.py / 3.txt)
- **SSH Сервер (IP):** `88.210.29.61`
- **SSH Порт:** `34002`
- **Пользователь:** `sokolovanv`
- **Пробрасываемые порты:**
  - `11434` (локальный) -> `11434` (удаленный, где крутится Ollama)
  - `2222` (локальный) -> `22` (удаленный, для SSH)

## Как поднять туннель в приложении Katya (после внедрения функционала)

1. Откройте вкладку **"Серверы"** (в самом низу настроек).
2. Заполните данные для подключения:
   - **IP-адрес:** `88.210.29.61`
   - **SSH Порт:** `34002`
   - **Пользователь:** `sokolovanv`
   - **Пароль:** *<введите ваш пароль>*
3. Нажмите **"Сохранить настройки"**.
4. В разделе "Настройка SSH-туннеля":
   - **Локальный порт:** `11434`
   - **Удаленный порт:** `11434`
5. Нажмите **"Поднять туннель"**.

## Настройка Local API AI для работы через этот туннель
1. Откройте раздел **"Модели ИИ"** -> **"Локальный API"**.
2. Укажите адрес: `http://127.0.0.1:11434`
3. Убедитесь, что модель в настройках соответствует той, что загружена на сервере (например, `llama3`).
